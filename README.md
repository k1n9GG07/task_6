## ✅ Выполненные этапы

### Этап 1 — Реализация архитектурной схемы

Разработана **единая архитектурная схема** для всего программного решения, включающая 4 уровня:

- **Клиентский уровень** — браузер пользователя, React SPA (основной интерфейс) и Vue Admin Panel (панель администратора), CDN для раздачи статики
- **Уровень шлюза** — Nginx в роли Reverse Proxy с SSL-терминацией и балансировкой нагрузки
- **Серверный уровень** — Node.js + Express REST API с JWT-авторизацией и ORM (Sequelize)
- **Уровень данных** — PostgreSQL (основное хранилище) + Redis (кеш и сессии)

**Обоснование единой схемы:** все компоненты являются частью одного программного продукта, связаны единым потоком данных и разворачиваются в рамках одной инфраструктуры. Раздельные схемы применяются только при наличии независимых, изолированных программных продуктов.

### Этап 2 — Описание технических требований к элементам схемы

Для каждого из 8 элементов архитектурной схемы составлена детальная таблица с указанием:

- **Аппаратных требований** — минимально необходимый объём ОЗУ, количество процессорных ядер, дисковое пространство
- **Программных требований** — конкретные версии ОС, фреймворков, библиотек и инструментов

Пример ключевых элементов:

| Элемент | ОЗУ | Ключевые компоненты |
|---------|-----|---------------------|
| Node.js API | 4 ГБ | Node.js 20 LTS, Express 4, JWT, Sequelize |
| PostgreSQL | 4 ГБ | PostgreSQL 16, 100 ГБ SSD |
| React SPA | 2 ГБ | Node.js 20, Vite 5, React 18, Redux Toolkit |
| Nginx | 1 ГБ | Nginx 1.24+, SSL/TLS, OpenSSL |
| Redis | 2 ГБ | Redis 7.x |
